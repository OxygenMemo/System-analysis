Output

ส่วนใหญ่จะเป็นพวก หน้าจอ
หน้าจอส่วนใหญ่ต้องนำข้อมูลมาแสดง
แต่ ข้อมูลที่นำมาแสดงจะขึ้นอยู่กับ Actor ว่า Actor นั้นๆ ควรเห็นอะไร

ประเภท output

  • hardcopy : พวกกระดาษ
  • softcopy : file
การออกแบบ output ยากตรงที่ว่าใครจะดูแล้วเค้าจะดูในรูปแบบอะไร ต้องตอบโจทย์เค้า

การออกแบบ output

เราก็ไปขอเอกสารเค้ามาที่เค้าใช้ทุกวัน มาสร้างเป็น output
(มันจะมี user แบบอยากเห็นสรุปแต่ไม่พิมรายระเอียด) เราก็ต้องดูรายงานว่าข้อมูลนั้นๆ เราเก็บไปหรือยัง
*มันจะวนกลับไปกลับมา*

TIP

output ต้องตอบโจทย์องกร บรรุวัตถุประสงองกร (การออกแบบ ตาม แพทฟอมเป็นแค่ตัวเลือก)

ตัวอย่าง

มหาวิทยาลัยบูรพา มีเป้าหมายคือ ผลิตบันดิท ที่มีคุณภาพ ต่อสังคม เค้าก็ต้องการจะเห็นอัตรานิสิตจบได้งาน

  • จริงๆ แล้ว User เค้าไม่รู้หลอกว่าเค้าต้องการอะไร จนกว่าเค้าจะได้ลองใช้
  • เราก็จะเจอคนที่ รอออกแบบสมบูรแล้วค่อยทำ จริงๆแล้วเป็นไปไม่ได้

External Output

ออกไปให้คนนอกองกรเห็น

คือ อะไรที่ส่งออกไปข้างนอก คือ External Output เช่น

  • Utility bills
  • Advertisements :โฆษนา
  • Payment : เช็ค

Internal Output

ใช้กันในองกรพวก รายงานบางอย่าง เช่นยอดขาย

  • Summary report : อะไรที่เกิดการบวกลบคูนหาร รวม
  • Detailed report : ต้องการดูรายระเอียด เช่น user ต้องตรวจสอบสิ่งที่เค้า Key เข้าไปได้
    (ต้องทีรายระเอียด Summary จะตามมา)
  • Historcal report : ประหวัติ ต้องเลือกช่วงข้อมูลได้ เช่น ระบบทะเบียน Reg มีข้อมูล 5 ล้านแถว 38-60 แล้วมันจะมี filed นึงเก็บปี เพื่อเอาไว้ select เพราะ เวลาเลือกเราจะเจอข้อมูลแน่ๆ กะไม่ให้เจอ Not Found (แล้วแต่เทคนิค) อารมแบบ จริงๆมีข้อมูล 5ล้านแถว เลยเก็บแยกปี เช่น สร้างตารางปี แล้ว ทำ reation กับ user เพื่อเก็บแยกกัน ไม่ให้ 5 ล้านแถวมารวมกัน เพราะเวลา select จะต้องเลือก จาก 5 ล้าน แต่เปรี่ยนเป็นเลือกจากปีแทน
  • Exception report : คือ การงานสิ่งผิดปกติ เช่นคนไม่จ่างเงิน , หนังสือขายดีผิดปกติ
    (ก็แล้วแต่ที่เราสนใจ)

เช่น ตัวอย่าง พนักงาน 7-11 เค้าเลิกกะ ต้องพิม report ตัวใหน มันคือ ยอดขาย = Summary report

เช่น บริษัท มีพนักงาน 30 คน ทำ ot 10 เค้าต้องการดู คนทำ ot เราควรแสดง หมด 30 คน (เพื่อใม่ให้ข้อมูลหาย)

วิธีแยก report

    Summary report : คือรวมมาแล้ว

Green IT

คือเช่น ค่าประกาศในการประชุม 1 ปี ซื้อ BM ได้ 1 คัน

คำคมอาจาร

  • อย่าไปหวังว่า user จะบอกเราได้ว่าเค้าต้องการอะไรตั้งแต่แรก
  • คำสั่ง SQL จะทำอะไรง่ายๆ ได้ แต่ แต่ละ DBMS จะมี option ยากๆ เป็นของตัวเอง
  • ศัพท์ turnaround report พวกค่าน้ำค่าไฟ ปริ้นไป เค้าต้องเอากลับมาจ่ายอยู่ดี
  • ศัพท์ Creet Stupit : คนโง่ก็ใช้ได้

สรุป

  • report มันคือการเอาข้อมูลมาแสดงเป็นตัวอักษร
  • Summary report : แสดงข้อมูลรวมเป็นแถวๆ สิ่งที่แสดงออกมาต้องตรงความต้องการของ User
  • การออกแบบหน้าจอ บางที่เราก็ไม่รู้ว่าต้องป้อนอะไร แปลว่าเราขาดคำอธิบาย
  • การออกแบบเราต้องคำนึงถึง UX เราเป็นคนสร้างเราก็รู้ว่ามันใช้ง่าย
  • คีข้อมูลให้ถูกต้อง และ ข้อมูลต้องเข้า DB ได้ถูกต้องด้วย
  • การออก แบบ input ต้อง คำนึงถึง User ไม่ให้เค้าเข้าใจผิด (ให้คนอื่นช่วย Test )
